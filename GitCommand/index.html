Git基础（1）

一、查看Git版本

git --version

二、查看计算机里有哪些版本的Git版本，并选用你想要的版本(后三步看不懂)

1.查看有哪些版本

which -a git

2.配置.bash_profile文件，选择自己需要的版本

vim .bash_profile

3.加入需要制定环境相对路径

export PATH="/user/local/git/bin:$SPAH"

4.编辑好后，重新载入.bash_profile

source .bash_profile

三、配置自己的Git信息

1.配置用户名

git config --global user.name wangjunli

2.配置邮箱

git config --global user.email 13@qq.com

四、配置的三个级别

1.system --针对于系统

2.global --针对于当前用户

3.local --针对于当前仓库

配置的优先级：local>global>system

五、如何查看Git文档

1.git config --help

2.git help config

六、重命名Git命令

1.git config --gloabl alias.co checkout

2.git config --gloabl alias.br brach

3.git config --gloabl alias.st status

4.git config --gloabl alias.ci commit

Git本地分支和分支合并(2)

分支具有协作功能（我们将做好的项目放到主分支上，如果想再添加新的功能时，可以放在一个新添加的分支上，等项目开发完，再将这个分支合并到主分支上）。

一、Git本地创建分支

1.新建test分支

git branch test

2.提交test分支到远程仓库

git push origin test:test

3.查看分支命令

git branch(不带参数，列出本地已经存在的分支，而且会在分支前面加*号标记。)

4.列出远程分支

git branch -r

5.列出本地分支和远程分支

git branch -a

6.重命名分支

git branch -m|-M oldbranch newbranch

7.删除本地分支

git branch -d|-D branchname

8.删除远程分支

git branch -d -r branchname

二、Git给分支起别名（打标签）

git标签分为两种：轻量标签和附注标签

轻量标签是指向提交对象的引用

附注标签则是仓库中的一个独立对象。建议使用附注标签。

1.创建轻量标签

git tag v0.1.2 -light（创建轻量标签不需要传递参数，直接制定标签名称即可。）

2.创建附注标签

git tag -a v0.1.2 -m "0.1.2版本"（创建附注标签时，参数a即annotated的缩写，制定标签类型，后附标签名。参数m指定标签说明，说明信息会保存在标签对象中）

3.查看历史版本示意图

git log --oneline --decorate --graph --all

注意：用Git命令起名为“v0”，并表示他的HS值。如果不制定Hash的话，会默认制定HEAD指定的Hash（也就是当前HS），一般我们只用5-7位的Hash。

三、分支之间的切换

1.切换到test分支上

git checkout test

2.创建+切换到test分支上

git checkout -b test

四、合并分支

合并某分支到当前分支（eg合并dev到master）

git merge dev

Git远程协作命令(3)

一、克隆远程仓库到本地

git clone https://github.com/H2OCO2/H2OCO2.github.io.git

注意：本地不需要再建仓库了

二、git push命令

将本地分支的更新，推送到远程仓库主机上。

git push <远程主机名> <本地分支名>:<远程分支名>

注意：分支推送顺序的写法是<来源地>:<目的地>,

git pull是<远程分支>:<本地分支>

git push是<本地分支>:<远程分支>

如果省略本地分支名，则表示将本地分支推送与之存在“追踪关系”的远程分支（通常两者同名），

如果该远程分支不存在，则会被新建。

1.本地的master分支推送到origin主机的master分支。如果远程主机上master分支不存在，则会被新建。

git push origin master

2.若省略本地分支名，则表示删除指定远程分支，因为这等同于推送一个空的本地分支到远程分支。

git push origin :master

git push origin ---delete master

上面两行代码意思相同，表示删除origin主机的master分支。

3.如果当前分支与远程分支之间存在追踪关系，则本地分支和远程分支都可以省略。

git push origin(将当前分支推送到origin主机对应的分支)

4.如果当前分支只是一个追踪分支，则主机名也可以省略。

git push

5.推送Tags

git push --tags

6.用push删除远程分支

git push --delete origin feacher

Git代码复原(4)

1.显示提交的log

git log -3

2.回滚到指定版本

git reset --hard commit码

3.强制提交

git push -f origin master
